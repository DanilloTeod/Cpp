##### CONFIGURAR CHAVE SSH #####

ssh é um protocolo para comunicação de dados com segurança
A ideia básica é cadastrar previamente quais computadores podem acessar o Github em seu nome. Outros computadores não conseguem acessar

(1) Gerar chave SSH no seu pc
(2) Cadastrar a chave no github

https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent

ssh-keygen -t ed25519 -C "email"

Salvo a chave no dir, entrar no arquivo .pub e colar o conteúdo no Github "SSH and GPG Keys"

#### SALVAR A PRIMEIRA VERSÃO ####


git init 

git add .    // enviar arquivos do projeto para uma area STAGE 

git status // VER O STAGE

git commit -m "Mensagem explicativa" // salva a versão

git branch -M main // salva histórico de commit

// deve-se criar um projeto no github

git remote add origin git@github.com:seuusuario/seurepositorio.git // associa dados local no github

git push -u origin main

git pull  // git fetch + git merge. Ele baixa os dados e já tenta aplicar no seu código na hora.

git clone URL // Baixa um projeto inteiro do GitHub para sua máquina.

git log // Exibe o histórico de todos os commits realizados.

git diff // Mostra as diferenças entre o arquivo que eu tenho e o último commit
// para sair apertar "q"

git restore nomedoarquivo // voltar para a última versão

git branch // mostra a branch que estou atualmente

git checkout -b nomedaminhabranch // saindo da branch que estou atualmente, cria outra e jogando todas as alterações para a nova branch 

git checkout main // volta para a branch

git checkout medaminhabranch

Mandar as alterações que fiz na minha branch separada para a branch principal

git merge nomedaminhabranch 

#### REPOSITÓRIOS REMOTOS ####

Pegar nosso historico de versionamento e levar para um lugar remoto (nuvem)

git fetch // Baixa as novidades do servidor, mas não mexe no seu código atual. Ele apenas atualiza o histórico para você ver o que os outros fizeram.



#### .gitignore #### 

Nem tudo o que está na sua pasta deve ir para o GitHub. Arquivos de senha, 
chaves de API, ou pastas gigantes de dependências (como node_modules no JS ou venv
 no Python) devem ficar de fora.

.env           # Ignora arquivos de senhas
node_modules/  # Ignora pastas pesadas
*.log          # Ignora qualquer arquivo que termine em .log


1. Obter a URL do projeto
Vá até a página do seu repositório no GitHub, clique no botão verde " Code" e copie o endereço (se configurou SSH como nas suas anotações, use a aba SSH; se não, use HTTPS).

2. Trazer o projeto para o seu PC (Clone)
Abra o terminal na pasta onde você organiza seus códigos e digite:
git clone URL_QUE_VOCE_COPIOU

O que o clone faz? Ele cria uma pasta com o nome do projeto, baixa todos os arquivos, todo o histórico de commits e já configura o remote origin automaticamente para você.

3. Entrar na pasta
cd nome-do-repositorio
(Muitas pessoas esquecem esse passo e tentam rodar comandos do git fora da pasta do projeto).

4. Sincronizar (Garantir que está na versão mais atual)
Se você não mexe nesse projeto há algum tempo, pode ser que existam alterações no GitHub que não estão no seu PC. Por garantia, rode:
git pull origin main
